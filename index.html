Aplicado a productos de formación telemática con recursos audiovisuales interactivos

Departamento de Prácticas QA · Edición 2026

1. Introducción

Este documento recoge las principales herramientas de testing utilizadas en el desarrollo de productos de formación empresarial telemática con recursos audiovisuales interactivos.

Principales herramientas de testing a explorar en esta jornada:

  • PHPUnit, Pest PHP, Codeception y Selenium WebDriver para testing en PHP/Symfony.
  • Vitest, Vue Testing Library, Cypress y Playwright para testing en Vue.js.
  • Jest y DOM Testing Library para testing de lógica e interfaz en JavaScript.
  • Navicat, utPLSQL, pgTAP y tSQLt para validación y testing en bases de datos.

Los productos requieren testing continuo en:

  • Interactividad y gamificación.
  • Animaciones y recursos audiovisuales.
  • Compatibilidad entre navegadores y dispositivos.
  • Rendimiento y estabilidad.
  • Integración backend–frontend.
  • Validación de datos, estados y progreso del usuario.

2. Testing para PHP Simphony (backend de la plataforma)

Symfony es un framework que permite construir aplicaciones web en PHP. En una empresa, el backend (la parte que procesa datos, reglas y APIs) debe ser estable, seguro y predecible. Para garantizarlo, se utilizan herramientas de testing automatizado, que ejecutan pruebas sin intervención humana. A continuación se explican las herramientas más usadas, desde las más básicas hasta las más avanzadas.

2.1. PHPUnit

Qué es: PHPUnit es la herramienta estándar para hacer pruebas en PHP. Piensa en PHPUnit como un “robot” que ejecuta pequeñas funciones del sistema y comprueba que devuelven lo que deberían.

Uso en la empresa: Validación de lógica de negocio, APIs internas, controladores y servicios.

Ventajas:

  • Estándar en PHP: todas las empresas lo conocen.
  • Ideal para estabilidad: detecta errores antes de que lleguen a producción.
  • Perfecto para CI/CD: se integra fácilmente en pipelines automáticos.

Desventajas:

  • Verboso: escribir tests complejos puede requerir mucho código.
  • No cubre UI: no sirve para probar pantallas, botones o animaciones.

2.2. Pest PHP

Qué es: Pest es una alternativa moderna a PHPUnit. Hace lo mismo, pero con una sintaxis más simple y fácil de leer. Piensa en Pest como una versión “minimalista” de PHPUnit.

Uso en la empresa: Testing rápido y legible para módulos Simphony.

Ideal para equipos que quieren escribir tests sin tanta complejidad.

Ventajas:

  • Sintaxis clara: los tests parecen frases, no bloques de código complicados.
  • Minimalista: menos ruido, más enfoque en lo importante.
  • Perfecto para proyectos nuevos: se empieza más rápido.

Desventajas:

  • Comunidad más joven que PHPUnit: menos documentación y ejemplos que PHPUnit.

2.3. Codeception

Tipo: E2E, API, integración

Qué es:Codeception es un framework de testing “todo en uno”. Permite probar todo el sistema completo, desde el backend hasta el frontend

Imagina que Codeception es un “tester profesional” que puede:

  • enviar peticiones a APIs,
  • simular usuarios navegando,
  • probar bases de datos,
  • ejecutar flujos completos.

Uso en la empresa:

  • Testing de flujos completos entre backend y frontend.
  • Validación de procesos gamificados con varias capas (API + UI + lógica).

Ventajas:

  • Framework “todo en uno” (API, UI, integración).
  • Compatible con Selenium y Playwright: puede controlar navegadores reales, donde hay muchas interacciones.
  • Ideal para plataformas gamificadas con múltiples capas (Complejas)donde hay muchas interacciones.

Desventajas:

  • Configuración avanzada: requiere más tiempo y conocimiento.

2.4. Selenium WebDriver

Tipo: Automatización de UI (End-to-End)

Qué es: Selenium es una herramienta que controla un navegador real (Chrome, Firefox, etc.) como si fuera un usuario humano.

Puede:

  • hacer clic,
  • escribir,
  • mover el ratón,
  • cambiar de pestaña,
  • interactuar con animaciones, videos y elementos visuales.

Uso en la empresa:

  • Testing de interfaces Complejas, animaciones y contenido audiovisual.
  • Validación de experiencias gamificadas donde la interacción es clave.

Ventajas:

  • Estándar de la industria: usado por empresas grandes.
  • Multinavegador: funciona en Chrome, Firefox, Edge, Safari.
  • Control total del navegador: múltiples pestañas, iframes, ventanas.
  • Integrable con Codeception y otros frameworks, complementandose muy bien.

Desventajas:

  • Configuración más compleja que Cypress o Playwright : requiere instalar drivers y configurarlos.
  • Scripts más verbosos para perfiles junior: mas código para hacer cosas simples.
  • Depende de drivers externos:ChromeDriver, GeckoDriver, etc.

Demo de código (PHP/Symfony con PHPUnit)

// tests/Service/ScoreServiceTest.php
use PHPUnit\Framework\TestCase;

final class ScoreServiceTest extends TestCase
{
    public function testCalculaPuntosPorAciertos(): void
    {
        $service = new ScoreService();

        $puntos = $service->calcular(8, 10);

        $this->assertSame(80, $puntos);
    }
}

3. Testing para Vue.js (frontend interactivo)

Vue.js es una tecnología que permite construir la parte visual de una aplicación web: botones, animaciones, pantallas, formularios, juegos y cualquier elemento con el que el usuario interactúa. En productos de formación gamificada, esta parte es esencial: el alumno hace clic, avanza, desbloquea contenido, gana puntos y recibe retroalimentación visual.

Para asegurarnos de que todo esto funciona correctamente, usamos herramientas de testing automatizado. Estas herramientas simulan usuarios, acciones y situaciones reales, comprobando que la interfaz responde como debería.

3.1. Vitest

Tipo: Unit & Integration

Qué es: Vitest es una herramienta que prueba pequeñas partes de la interfaz de forma rápida. Imagina un robot que revisa cada botón, cada cálculo y cada reacción del sistema para asegurarse de que todo funciona como se espera.

¿Qué significa testing unitario en Vue?

Probar una pieza pequeña del sistema, como:

  • Un botón que suma puntos.
  • Un componente que muestra una tarjeta.
  • Una función que calcula el nivel del usuario.

Es como revisar un ladrillo individual de una pared.

¿Qué significa testing de integración en Vue?

Probar varias piezas juntas, por ejemplo:

  • Un formulario que actualiza el progreso del alumno.
  • Una pantalla que combina animaciones y datos.
  • Un componente que depende de otro para funcionar.

Es como comprobar cómo encajan varios ladrillos entre sí.

Uso en la empresa:

  • Testing de componentes interactivos.
  • Validación de lógica de gamificación (puntos, niveles, recompensas).
  • Comprobación de estados (clics, avance, retroceso).

Ventajas:

  • Muy rápido.
  • Perfecto para Vue.
  • Ideal para lógica interna.

3.2. Vue Testing Library

Tipo: Unit & Integration

Qué es: Vue Testing Library prueba componentes como si fueran usados por una persona real. En lugar de centrarse en el código, se centra en cómo se ve y cómo se usa la interfaz.

Uso en la empresa:

  • Testing de comportamiento real.
  • Validación de interacciones (clics, escritura, selección).
  • Comprobación de accesibilidad.

Ventajas:

  • Simula el uso real del usuario.
  • Compatible con Vitest.
  • Ideal para componentes visuales.

Desventajas:

  • Más lento que Vitest.
  • Menos control interno del código.

3.3. Cypress

Tipo: End-to-End

Qué es: Cypress abre un navegador real y simula un usuario completo: navega, hace clic, escribe, espera animaciones y comprueba que todo funciona como en producción.

Uso en la empresa:

  • Testing de flujos completos.
  • Validación de interacciones reales.
  • Detección de errores visuales.

Ventajas:

  • Debug visual.
  • Ideal para E2E.
  • Detecta errores reales.

Desventajas:

  • Más lento que Vitest.
  • Limitaciones con múltiples pestañas.

3.4. Playwright

Tipo: End-to-End avanzado

Qué es: Playwright es similar a Cypress, pero más potente. Permite probar varios navegadores, dispositivos, pestañas y ventanas.

Uso en la empresa:

  • Testing de interfaces complejas.
  • Validación en distintos dispositivos.
  • Pruebas de rendimiento.

Ventajas:

  • Multinavegador real.
  • Muy rápido y estable.
  • Perfecto para escenarios avanzados.

Desventajas:

  • Más técnico.
  • Configuración avanzada.

Demo de código (Vue con Vitest + Vue Testing Library)

// QuizCard.test.js
import { render, screen } from '@testing-library/vue';
import { describe, it, expect } from 'vitest';
import QuizCard from './QuizCard.vue';

describe('QuizCard', () => {
  it('muestra el progreso actual', () => {
    render(QuizCard, { props: { progreso: 60 } });

    expect(screen.getByText('Progreso: 60%')).toBeInTheDocument();
  });
});

4. Testing para JavaScript (lógica interactiva)

JavaScript es el lenguaje que controla la lógica interactiva de una aplicación web: animaciones, botones, cálculos, estados, temporizadores y cualquier acción que ocurre cuando el usuario interactúa con la pantalla.

4.1. Jest

Tipo: Unit testing

Qué es: Jest es una herramienta que prueba funciones y lógica interna. Imagina un robot que revisa cada cálculo, cada regla y cada decisión que toma el sistema.

Uso en la empresa:

  • Testing de funciones internas.
  • Validación de cálculos de gamificación.
  • Comprobación de estados y reglas.

Ventajas:

  • Muy maduro.
  • Gran comunidad.
  • Ideal para lógica interna.

Desventajas:

  • Más lento que Vitest.

4.2. DOM Testing Library

Tipo: Unit/Integration

Qué es: DOM Testing Library prueba cómo se comporta la interfaz desde el punto de vista del usuario. Se centra en accesibilidad, interacción y experiencia real.

Uso en la empresa:

  • Testing de accesibilidad.
  • Validación de comportamiento visual.
  • Simulación de interacciones reales.

Ventajas:

  • Enfoque en UX.
  • Compatible con Jest y Vitest.

Desventajas:

  • No cubre E2E.

Demo de código (JavaScript con Jest)

// score.test.js
import { calcularNivel } from './score';

test('asigna nivel Oro con 1000 puntos o más', () => {
  expect(calcularNivel(1200)).toBe('Oro');
});

test('asigna nivel Bronce por debajo de 500 puntos', () => {
  expect(calcularNivel(320)).toBe('Bronce');
});

5. Testing para Bases de Datos (Navicat y frameworks SQL)

Las bases de datos almacenan toda la información importante: progreso del alumno, puntuaciones, módulos completados, estadísticas y contenido. Probar una base de datos significa asegurarse de que los datos se guardan, se leen y se actualizan correctamente.

5.1. Navicat

Tipo: Validación manual y automatización ligera

Qué es: Navicat es una herramienta visual que permite ver y editar bases de datos. Es como un panel de control donde puedes comprobar que los datos están bien guardados.

Uso en la empresa:

  • Validación manual de datos.
  • Comprobación de integridad.
  • Revisión de tablas y registros.

Ventajas:

  • Muy fácil de usar.
  • Ideal para QA manual.

Desventajas:

  • No automatiza tests unitarios.

5.2. utPLSQL / pgTAP / tSQLt

Tipo: Unit testing SQL

Qué son: Son herramientas que permiten probar funciones internas de la base de datos. Imagina que la base de datos también tiene “pequeños robots” que realizan cálculos y validaciones. Estas herramientas prueban que esos robots funcionan bien.

Uso en la empresa:

  • Testing de funciones SQL.
  • Validación de triggers.
  • Comprobación de reglas internas.

Ventajas:

  • Testing profundo.
  • Ideal para bases de datos complejas.

Desventajas:

  • Requiere scripting avanzado.

Demo de código (Base de datos con pgTAP)

-- tests/progreso_usuario_test.sql
SELECT plan(2);

SELECT is(
  (SELECT puntos_totales FROM progreso_usuario WHERE usuario_id = 10),
  950,
  'El usuario 10 conserva sus puntos esperados'
);

SELECT ok(
  EXISTS(SELECT 1 FROM progreso_usuario WHERE modulo_id = 4),
  'Existe progreso registrado para el módulo 4'
);

SELECT * FROM finish();

6. Conclusión

El testing es una parte esencial del desarrollo de productos de formación gamificada. Cada herramienta tiene su propósito y su lugar en el flujo de trabajo. La combinación adecuada de estas herramientas garantiza que los productos sean estables, interactivos y satisfactorios para los usuarios finales.
